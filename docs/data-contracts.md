# 数据契约

所有时间采用北京时间 ISO 8601，日期采用 `YYYY-MM-DD`，货币固定 CNY，经纬度固定高德 GCJ-02。

- `UserProfile`：长期非敏感默认值。
- `TripRequest`：一次规划的完整、已标准化输入。
- `SourceEvidence`：来源、工具、查询时间、实时性和验证状态。
- `PoiCandidate`：唯一 POI ID、名称、地址、经纬度、类别和证据。
- `HotelCandidate`：酒店 POI、位置说明、平台搜索入口和强制确认提示。
- `Activity`：日期、时间、POI、费用、预约信息、锁定状态。
- `RouteLeg`：相邻活动、方式、距离、时长、摘要、导航入口和证据。
- `Itinerary`：请求、概览、每日活动、路线、酒店、天气、预算、警告和验证状态。
- `ValidationIssue`：级别、代码、信息、日期/活动关联和修复建议。

模型输出只允许提交草案字段；证据、验证状态、预算合计和路线信息由程序覆盖，禁止模型自报“已验证”。
