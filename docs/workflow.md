# 工作流、降级与验收

## 固定顺序

启动检查 -> 输入标准化 -> 目的地/天气 -> POI -> 酒店 -> DeepSeek 草案 -> 高德路线 -> 时间重排 -> 确定性校验 -> 渲染 -> 事务保存 -> ICS。

共享状态步骤必须串行；只有互不依赖的 POI 分类查询可并行。任一步骤均返回结构化状态、警告和错误，不以自然语言关键词推断成功。

## 验证状态

- `draft`：尚未完成高德路线与校验。
- `unverified`：高德不可用或关键证据缺失。
- `verified`：所有必需活动有 POI 证据，相邻路线完整且无错误级校验项。

## 验收基线

- 北京三日样例无跨日、重叠和路线断点。
- 家庭、老人、儿童、低预算、自驾和公交优先输入可被标准化。
- DeepSeek、高德、单条路线和 SQLite 故障均产生对应降级状态。
- 酒店不出现未经正式来源验证的价格、库存或取消承诺。
- ICS 使用真实活动开始/结束时间，全天事件结束日期为次日。
