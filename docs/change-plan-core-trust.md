# 修改方案：基础核心数据契约与可信校验

> 状态：实现完成，草稿 PR #3 已创建且 Python CI 通过（用户于 2026-07-19 明确确认）

## 1. 目标与非目标

- 目标：建立离线可测试的证据生命周期、敏感信息门禁、确定性校验和 SQLite schema 迁移，使未经真实工具证明的数据不能进入 `verified`。
- 用户场景：在未配置 Node/高德时可在 DeepSeek 可用的前提下生成醒目标注的未验证草案；完全离线环境使用假模型、假 MCP 验证相同降级路径，并可靠保存非敏感本地数据。
- 成功标准：过期、来源错误、未知 POI、不可达路线、锁定活动变化和敏感输入均被代码拦截；旧 SQLite 可备份迁移；全部离线测试通过。
- 本次不做：不安装 Node，不配置或调用真实 Key，不实现完整十阶段状态机、真实高德工具证据适配器、完整 ICS/UI 改造或任何外部写操作。

## 2. 规范影响

- 是否需要修改 `limitation.md`：否；本次实现已有规则，不改变产品边界。
- 是否新增 MCP、数据源或外部写操作：否。
- 是否涉及敏感数据：实现检测与拒绝，不新增保存类型。
- 数据可信度与来源：没有可审计工具调用记录时，高德相关数据统一为 `unverified`。

## 3. 数据契约与设计

- 增加 `expired` 和证据类型；证据保存北京时间查询时间、过期时间、原始标识符和坐标系。
- 默认 TTL：POI 位置 30 天、营业状态 24 小时、当前天气 3 小时、路线 6 小时、酒店位置 7 天；预报必须显式提供供应商期限。
- 状态优先级：必需阶段未完成为 `draft`；来源或业务错误为 `unverified`；仅有过期问题为 `expired`；全部有效且无错误才为 `verified`。
- 输入边界：城市 1–64 字符，列表最多 30 项且单项 1–80 字符，城际交通最多 1000 字符，儿童年龄 0–17，预算上限与 UI 一致，外部 DTO 禁止额外字段。
- 敏感信息门禁拒绝真实密钥/私钥、Cookie/密码、身份证、银行卡、完整手机号和明确标注的家庭地址；只返回字段与类别，不记录原文。
- `ValidationContext` 提供固定时间、目的地确认、批准 POI、锁定活动快照和必需阶段完成状态，不再用一个 `amap_available` 布尔值提升结果。
- DeepSeek 与高德只接收经过规范化的最小 DTO；模型输出的 `verified` 不生效。

## 4. SQLite 迁移与降级

- 持久化根对象增加 `schema_version=1`、稳定 ID、创建/更新时间；行程增加 `run_id` 和非敏感请求摘要。
- 数据库使用 `PRAGMA user_version=1`，对旧版 `0` 库执行显式事务迁移。
- 迁移前通过 SQLite backup API 在数据库相邻的 `backups` 目录创建 `.db` 备份；不存在旧数据库时直接创建 v1。
- 迁移或回读失败必须回滚并保留原库与备份；应用继续提供会话态结果，但明确显示未保存，不写入其他位置。
- 回滚代码变更时使用新的 PR；数据回滚使用迁移前备份替换本地数据库，操作前由用户确认。
